# B2.3 - Matriz de validacao Word (.doc/.docx)

## Objetivo
Validar comportamento da ingestao Word em 4 cenarios operacionais da Etapa B2.

## Matriz esperada

| Cenario | Entrada | Configuracao | Resultado esperado |
|---|---|---|---|
| 1 | `.docx` | padrao | `OK` + `Status: DOCX_DIRETO` |
| 2 | `.doc` | conversao ON + fallback ON | `OK` + `Status: DOC_CONVERTIDO` (se LibreOffice disponivel) ou `DOC_FALLBACK` |
| 3 | `.doc` | conversao OFF + fallback ON | `OK` + `Status: DOC_FALLBACK` |
| 4 | `.doc` | conversao ON + fallback OFF + path invalido | `ERRO` (sem fallback) |

## Script de validacao automatizada
Arquivo: `scripts/validate-word-b23.ps1`

Exemplo (somente DOCX):

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\validate-word-b23.ps1 -DocxPath "C:\caminho\arquivo.docx"
```

Exemplo completo (DOCX + DOC):

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\validate-word-b23.ps1 -DocxPath "C:\caminho\arquivo.docx" -DocPath "C:\caminho\arquivo.doc"
```

## Observacoes
- O script faz backup e restauracao de `expertdev.properties` durante os testes.
- Se `-DocPath` nao for informado, os cenarios 2, 3 e 4 sao pulados.
- O runner usado nos testes e `br.com.expertdev.service.WordIngestionProbe`.
